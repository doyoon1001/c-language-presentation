Programming Language

컴퓨터가 이해하는 언어

기계어 (Machine Language)
  • 컴퓨터가 직접 이해하는 유일한 언어
  • 0과 1로 구성된 이진수 코드
  • CPU 내부 스위치 회로 ON/OFF 제어
  • 하드웨어에 완전 종속됨
/* 기계어 예시 */
00001111 10111111 01000101
11111000 00001111 10111111
01001101 11111000 00000011
프로그래밍 언어의 위치

기계어를 사용할 수는 있으나 이진수로 프로그램을 작성해야 하기 때문에 아주 불편하다.
프로그래밍 언어는 자연어와 기계어 중간쯤에 위치하며, 컴파일러가 프로그래밍 언어를 기계어로 통역해준다.

Compiler

컴파일러의 역할

컴파일러(Compiler)는 인간(프로그래머)과 컴퓨터(기계) 사이의 통역사이다.
고급 언어로 작성된 소스 코드를 컴퓨터가 이해할 수 있는 기계어로 번역해준다.

C언어의 소개와 역사

Language Classification

프로그래밍 언어의 분류

기계어
  • 특정 CPU 명령어를 이진수로 표시
  • 하드웨어에 종속 — CPU마다 다름
  • 직접 실행 가능, 속도 최고
  • 인간이 읽고 쓰기 극히 어려움
어셈블리어
  • 이진수 대신 영어 기호(니모닉) 사용
  • 기호와 CPU 명령어가 1대1 대응
  • 어셈블러(Assembler)가 기계어로 변환
MOV AX, MIDSCORE
ADD AX, CX
MOV TOTALSCORE, AX
고급 언어
  • 특정 하드웨어와 무관하게 작성
  • 인간 언어에 가장 가까운 문법
  • 컴파일러가 기계어로 번역
  • C, C++, Java, Python, FORTRAN…
TotalScore = Mid + Final;
고급 언어의 종류
FORTRAN
1950년대 최초의 고급 언어. 수치 계산·과학 기술 계산에 특화.
COBOL
1959년. 비즈니스 사무 처리용. 구어체 문장 형태로 기술.
Python
1991년 귀도 반 로섬. 인터프리트형. AI·데이터 과학 분야에 널리 사용.
C
1970년대 초 데니스 리치. UNIX 운영체제를 위해 AT&T 벨 연구소에서 개발.
C++
1983년 스트로스트럽. C에 클래스·객체지향 개념 추가.
Java
1995년 제임스 고슬링. 선 마이크로시스템. 객체 지향 언어.
Features of C

C언어의 특징과 장점

✂️
간결성
키워드가 32개에 불과하여 문법 구조가 간단하다. 핵심만 담은 최소한의 언어 설계.
⚡
효율성
컴파일 후 빠른 실행 속도. 실행 속도가 중요한 시스템 프로그래밍에 최적화.
🔧
저수준 + 고수준
하드웨어를 직접 제어하는 저수준 프로그래밍과 추상화된 고수준 프로그래밍을 동시에 지원.
🌍
이식성 (Portability)
ANSI 표준을 따르면 다양한 플랫폼에서 동일하게 동작. 한 번 작성하면 여러 시스템에서 실행 가능.
🖥️
하드웨어 이해
C언어를 이해하면 컴퓨터 하드웨어가 어떻게 동작하는지 깊이 이해할 수 있다.
🌟
C++, Java의 공통 뿌리
C언어는 C++, Java의 문법 기반이다. C를 배우면 다른 언어 학습이 훨씬 쉬워진다.
Weakness & Future

C언어의 단점과 미래

단점
1

초보자에게 어려움 — 메모리 관리, 포인터 등 복잡한 개념이 많음

2

포인터(Pointer) 오용 — 잘못 사용 시 심각한 버그와 보안 취약점 발생

3

수동 메모리 관리 — malloc/free를 직접 관리해야 함 (메모리 누수 위험)

4

객체 지향 미지원 — 대규모 소프트웨어 구조화에 한계

C언어는 계속 사용된다
✓

C++, Java의 공통 기반 — C 학습 → 다른 언어 습득 용이

✓

실행 속도가 중요한 시스템 프로그램은 C로 구현

✓

임베디드 시스템에서 C언어가 주력 언어

임베디드 시스템이란?

특수 목적의 시스템으로, 컴퓨터가 장치 안에 내장된 시스템.
MP3 플레이어, 스마트폰, 세탁기, 자동차 ECU 등이 해당된다.

Algorithm

알고리즘이란?

정의

알고리즘(Algorithm)이란 문제를 풀기 위하여 컴퓨터가 수행하여야 할 단계적인 절차를 기술한 것이다.

빵 만들기 알고리즘 (예)
1

빈 그릇을 준비한다

2

이스트를 밀가루, 우유에 넣고 저어준다

3

버터, 설탕, 계란을 추가로 넣고 섞는다

4

따뜻한 곳에 놓아두어 발효시킨다

5

170~180°C 오븐에서 굽는다

1~10 합 구하기 알고리즘
방법 1 — 직접 더하기

1+2+3+···+10 = 55

방법 2 — 그룹핑

합이 11인 쌍 5개 → 5 × 11 = 55

방법 3 — 수식 활용 (최적)

n(n+1)/2 = 10×11/2 = 55

같은 문제도 알고리즘에 따라 효율이 크게 달라진다. 좋은 알고리즘 선택이 좋은 프로그램의 시작이다.

Algorithm Description

알고리즘 기술 방법

자연어 (Natural Language)

일상 언어로 절차를 기술하는 방법

① 전화번호부를 펼친다
② 이름을 찾을 때까지 페이지를 넘긴다
③ 이름을 찾으면 번호를 기록한다
④ 끝까지 없으면 없음을 알린다
장점: 이해 쉬움 단점: 모호할 수 있음
순서도 (Flowchart)

프로그램 논리 순서를 그림으로 표현

시작
↓
아이디 / 비밀번호 입력
↓
일치 여부 확인?
YES
↓
로그인 성공
NO
↓
오류 메시지 표시
의사코드 (Pseudo Code)

자연어보다 체계적, 프로그래밍 언어보다 덜 엄격한 표현

합계 ← 0
for i = 1 to 10
  합계 ← 합계 + i
end for
평균 ← 합계 / 10
print 평균
체계적 언어 독립적 코드와 유사
Mini Project

3개의 수 중에서 최대값 찾기

여러 개의 숫자가 저장된 리스트에서 가장 큰 숫자를 찾는 알고리즘. 정답을 찾으려면 리스트의 모든 숫자를 순서대로 비교해야 한다.

자연어 기술
1

리스트의 첫 번째 숫자를 가장 큰 숫자(largest)로 가정한다

2

리스트의 나머지 숫자를 차례로 비교한다

3

현재 숫자가 largest보다 크면 largest를 갱신한다

4

리스트가 끝나면 현재 largest를 반환한다

의사코드 / C언어
/* 의사코드 */
largest ← List[0]
for each item in List
  if item > largest
    largest ← item
return largest

/* C언어 */
int max = arr[0];
for(int i=1; i<n; i++)
  if(arr[i] > max)
    max = arr[i];
return max;

알고리즘의 핵심: 문제를 작은 단계로 분해하고, 각 단계를 명확하게 기술한다. 좋은 알고리즘 없이는 좋은 프로그램도 없다.
