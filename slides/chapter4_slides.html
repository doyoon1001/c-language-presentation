Problem Definition

문제 정의

문제 목표

사용자로부터 원의 반지름(r)을 입력받아
원의 넓이와 둘레를 계산한 뒤 출력하는 프로그램을 작성한다.

📐 넓이 = π × r²
📏 둘레 = 2 × π × r
사용 헤더
#include <stdio.h> — printf, scanf 사용
#include <math.h> — M_PI 상수 사용
입력 (Input)

float형 변수 r — 원의 반지름

scanf("%f", &r);
처리 (Process)

넓이 계산

M_PI * r * r

둘레 계산

2 * M_PI * r
출력 (Output)

계산된 넓이와 둘레를 printf로 화면에 출력

원의 넓이: %.6f 원의 둘레: %.6f
Flowchart

순 서 도

시 작
입력
반지름 r 입력
처리
넓이 = π × r² 계산
처리

입력 단계

scanf("%f", &r)로 사용자에게 반지름 값을 받는다. float형으로 소수점 입력이 가능하다.

처리 단계 — 수식

📐 넓이

M_PI * r * r

M_PI는 math.h에 정의된 π 상수

📏 둘레

2 * M_PI * r
출력 단계

printf와 %f 서식으로 계산 결과를 소수점 포함 출력한다.

Code Explanation

원의 넓이·둘레 코드 설명

변수 선언

float형 변수 r 선언 — 소수점이 있는 반지름 값을 저장합니다.

입력 처리

scanf("%f", &r) — %f 서식으로 float형 값을 입력받습니다.

수식 계산
  • M_PI — math.h에 정의된 원주율 상수 (3.14159…)
  • 넓이: M_PI * r * r
  • 둘레: 2 * M_PI * r

김도윤
팀 장

이번 과제를 통해 math.h 라이브러리를 처음 활용해 보았는데, 직접 π 값을 정의하지 않아도 M_PI 상수 하나로 정확한 계산이 가능하다는 점이 인상적이었습니다. 간단한 코드일수록 기초가 얼마나 중요한지 다시 한번 느꼈습니다.

팀 원

저번 계산기 프로그램에서 double형을 사용해 보았는데, 이번 과제에서 float형을 접하게 되어 두 자료형의 차이를 직접 조사해볼 수 있었습니다. 정밀도와 메모리 크기 측면에서 차이가 있다는 것을 알게 되었고, 상황에 맞는 자료형을 선택하는 것이 중요하다는 점을 배웠습니다.
