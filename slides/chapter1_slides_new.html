03. Definition

컴퓨터의 정의

현대적 정의

컴퓨터(computer)는 단순히 계산(compute)만 하는 기계가 아니다.

"프로그램(명령어들의 리스트)에 따라
데이터를 처리하는 기계"

계산기

정해진 기능만 수행한다.
기능을 변경할 수 없다.

컴퓨터

수행하는 기능을 쉽게 바꿀 수 있다.
범용적인 기계이다.

프로그램 = 작업지시서

프로그램은 컴퓨터에게 해야 할 작업의 내용을 알려주는 문서이다.

예: 두 수의 합 프로그램

1

사용자로부터 2개의 숫자를 입력받는다.

2

숫자들을 더한다.

3

결과를 화면에 출력한다.

04. Concept

만약 컴퓨터에 프로그램이 없다면?

프로그램 없는 컴퓨터

컴퓨터 하드웨어만 있고 프로그램이 없다면, 컴퓨터는 그저 약간의 열과 소음을 발생하는 쓸모없는 기계에 불과하다.

"윈도우즈"와 같은 운영 체제를 설치하고 추가로 여러 가지 응용 프로그램을 설치해야만 비로소 유용하게 된다.

왜 프로그램을 설치하게 했을까?

컴퓨터를 범용적인 기계로 만들기 위해서이다.
프로그램만 바꾸어 주면 다양한 작업을 할 수 있다.

소프트웨어 계층 구조
👤 사용자
↕
📱 애플리케이션
↕
⚙️ 운영 체제 (OS)
↓
🖥️ 하드웨어

각 계층은 아래 계층의 서비스를 사용하고 위 계층에 서비스를 제공한다

05. Example

프로그램의 예시

평균 계산 프로그램

3개의 숫자를 받아서 평균을 계산하는 프로그램의 명령어(instruction) 목록:

1

첫 번째 숫자를 x로 입력받는다.

2

두 번째 숫자를 y로 입력받는다.

3

세 번째 숫자를 z로 입력받는다.

4

a = (x + y + z) / 3.0 을 계산한다.

5

a의 값을 화면에 출력한다.

/* 평균 계산 — C언어 예시 */ float x, y, z, a; scanf("%f %f %f", &x, &y, &z); a = (x + y + z) / 3.0; printf("평균: %f\n", a);
핵심 개념

프로그램은 특정한 작업을 위한 작업 지시서라고 볼 수 있다.

작업을 지시하려면 명령어들을 순서대로 나열해야 한다.

명령어(Instruction)란?
  • 컴퓨터에게 전달하는 하나의 작업 지시
  • 프로그램은 수십~수백만 개의 명령어로 구성
  • 컴퓨터는 명령어를 순차적으로 실행한다
06. History

프로그램의 역사

최초의 프로그래밍 가능 기계

해석 기관(Analytical Engine) — 찰스 배비지(Charles Babbage)가 설계한 기계로, 수천 개의 기어·바퀴·축·레버 등이 증기로 작동하도록 설계되었다. 실제로 만들어지지는 못했다.

배비지 해석기관의 구성
  • 중앙 처리 장치 — 계산 담당 (mill)
  • 메모리 — 중간 단계 숫자 임시 저장 (store)
  • 출력 장치 — 출력 숫자를 나타내는 다이얼
  • 입력 장치 — 천공 카드
찰스 배비지
Charles Babbage (1791–1871)
"컴퓨터의 아버지"로 불리는 수학자·발명가. 차분 기관과 해석 기관을 설계했다.
시대적 맥락

현대 컴퓨터의 기본 구조(CPU·메모리·입출력)가
이미 19세기에 개념으로 존재했다.

07. First Programmer

최초의 프로그래머

에이다 러브레이스 (Ada Lovelace)

대문호 바이런(Byron)의 친딸로, 뛰어난 수학적 재능을 지녔다.

배비지의 해석 기관에 매료되어 해석 기관을 위한 프로그램을 개발하였다.

서브루틴(subroutine), 루프(loop), 점프(jump) 등 핵심적인 프로그래밍 기본 원리를 고안하였다.

1842년 베르누이 수 계산 알고리즘을 기술한 주석이 역사상 최초의 컴퓨터 프로그램으로 간주된다.

에이다 러브레이스
Ada Lovelace (1815–1852)
"숫자의 마술사
(Enchantress of Numbers)"
업적

그녀의 이름을 딴 프로그래밍 언어 Ada가 지금도 항공·군사 분야에서 사용된다.

08. Von Neumann

초기 컴퓨터와 폰 노이만 구조

초기 컴퓨터 ENIAC의 문제

프로그램이 스위치로 기억되었고, 프로그램을 변경할 때마다 그 많은 스위치들을 처음부터 다시 연결해야 했다. 매우 비효율적이었다.

폰 노이만 구조 (해결책)
  • 프로그램을 메인 메모리에 저장한다 → 쉽게 변경 가능
  • 메인 메모리에 저장된 프로그램에서 명령어들을 순차적으로 가져와서 실행한다.

09. Detail Level

얼마나 자세하게 지시해야 할까?

컴퓨터에게 명령할 때

컴퓨터는 상식이나 지능이 없기 때문에 아주 자세하고 구체적으로 일을 지시하여야 한다.

🤔

나쁜 예

"우유, 밀가루, 버터, 설탕, 계란을 적당히 섞어서 발효시킨 다음, 오븐에 적당히 굽는다."

"적당히"라는 표현을 컴퓨터는 이해할 수 없다.

✅

좋은 예

"우유 200ml, 밀가루 300g, 버터 50g을 계량하고, 180°C 오븐에서 25분 굽는다."

프로그래머의 역할

컴퓨터가 이해할 수 있는 수준으로 단계별, 구체적으로 작업을 지시하는 것이 프로그래머의 핵심 역량이다.

이를 위해 필요한 것:

  • 알고리즘 — 문제 해결 절차
  • 프로그래밍 언어 — 표현 도구
  • 논리적 사고 — 순서대로 생각하기
컴퓨터의 장점
  • 작업을 굉장히 빠르게 처리한다.
  • 정확하게 실행한다.
  • 몇 번을 반복해도 불평이 없다.
